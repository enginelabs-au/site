# 02 — Engines

The Engines are the productized service catalog. Each Engine spec sheet is a one-pager that doubles as (a) a landing page tile, (b) a Control Centre recommendation target, and (c) a portfolio substitute until real case studies exist.

## Spec sheet template

Every Engine page follows the same structure so the site reads as a product catalog, not a list of essays:

1. **One-sentence outcome** — what the buyer gets.
2. **The work it retires** — concrete role/tasks, not technology.
3. **Inputs** — exactly 3 bullets, in plain English.
4. **Outputs** — exactly 3 bullets, in plain English.
5. **Typical integrations** — named tools.
6. **What's included** — concrete deliverables tied to a package tier.
7. **What's not included** — explicit exclusions, drawn from the MSA / Addendum.
8. **Human review** — where the human-in-the-loop sits, per Addendum §4.
9. **Price band** — `A$X → A$Y, scoped in the Control Centre`.
10. **Typical timeline** — calendar-week range.
11. **Dogfood note** — where applicable, "we use this ourselves to do X."
12. **"Configure in the Control Centre"** CTA.

## Engines deliberately not offered

To honour the MSA exclusions (§3) and Addendum prohibitions (§5), there is no:

- **Decision Engine** for legal, medical, financial, credit, insurance, employment, immigration, housing or safety decisions.
- **Compliance Engine** for SOC2 / ISO / HIPAA / regulated certification work.
- **Security Engine** for pen testing, incident response or managed monitoring.
- **Scraping Engine** for restricted-platform data extraction.
- **Cold Outreach Engine** without consent/permission gating (the Outreach Engine exists, but only on permissioned data).

If a brief asks for any of these, the Control Centre declines politely and explains why, citing the relevant policy.
